# คู่มือและคำอธิบายโครงสร้างโค้ดเชิงลึก (Satellite Conjunction Analysis Walkthrough)

เอกสารนี้รวบรวม **คู่มือการใช้งาน (User Manual)** และ **คำอธิบายการทำงานของโค้ดอย่างละเอียด (Detailed Code Explanation)** สำหรับระบบ Satellite Conjunction Analysis System ที่ได้รับการเพิ่มฟีเจอร์คำนวณวงโคจรขั้นสูง

---

## 📖 1. คู่มือการใช้งานระบบ (User Manual)

ระบบถูกออกแบบมาให้เป็น Web Dashboard ที่แบ่งหน้าจอเป็น 2 ส่วนหลักคือ **Sidebar (แถบตั้งค่าด้านซ้าย)** และ **Main Display (หน้าจอแสดงผล)**

### 1.1 โหมดการทำงาน (App Mode)
ที่แถบตั้งค่าด้านซ้ายสุด คุณสามารถเลือกโหมดการทำงานได้ 2 โหมด:
1. **1-on-1 Analysis (วิเคราะห์แบบเจาะจง 2 ดวง):** ใช้สำหรับหาระยะเข้าใกล้ที่สุด (TCA) ระหว่างดาวเทียมหลักและดาวเทียมรอง
2. **Broad Catalog Scanner (สแกนหาภัยคุกคามทั้งหมด):** ระบบจะนำดาวเทียมหลักที่คุณตั้งไว้ ไปเทียบระยะกับดาวเทียมทุกดวงในแคตตาล็อก เพื่อกรองหาว่ามีดวงไหนที่เสี่ยงจะพุ่งชนบ้าง

### 1.2 การระบุข้อมูลดาวเทียม (Input TLE)
- หากใช้ **แคตตาล็อก**, สามารถเลือกชื่อดาวเทียมจาก Dropdown ได้เลย ระบบจะดึงฐานข้อมูล TLE ที่อยู่ในเครื่องมาให้
- ระบบจะแสดงป้ายกำกับว่า TLE นี้ใหม่หรือเก่าแค่ไหน (เช่น *ข้อมูลใหม่: 0.8 วัน*)
- หากเลือกกรอก **ข้อมูลเอง (Manual)** คุณจำเป็นต้องกรอก TLE บรรทัดที่ 1 และบรรทัดที่ 2 อย่างถูกต้องตามมาตรฐาน NORAD

### 1.3 การจำลองอดีต/อนาคต (Historical/Future Analysis Window)
- **วันที่ และ เวลา (UTC):** คุณสามารถเลือกจุดเริ่มต้นของเวลาที่ต้องการคำนวณ (Epoch)
- **ระยะเวลาประเมินผล:** เลื่อนสไลเดอร์เพื่อดูไปข้างหน้าตั้งแต่ 1 ถึง 72 ชั่วโมง
- **ความละเอียด (Timestep):** ยิ่งตั้งค่าน้อย (เช่น ทุกๆ 1 นาที) กราฟจะยิ่งแม่นยำ แต่จะใช้เวลาประมวลผลนานขึ้น

### 1.4 การตั้งค่าระบบป้องกัน (Covariance & Radius)
- **ความคลาดเคลื่อนของตำแหน่ง (Position Uncertainty - σ):** ใช้คำนวณโอกาสชน ยิ่งตั้งค่ามาก กราฟ Heatmap จะยิ่งเบลอและกินพื้นที่กว้างขึ้น
- **รัศมีวัตถุรวม (Combined Hard-body Radius):** รัศมีขนาดของดาวเทียมทั้งสองดวงรวมกัน (กม.) เพื่อใช้ดูโอกาสในการแตะกัน 

### 1.5 การจำลองการหลบหลีก (Maneuver Simulation)
คุณสามารถใส่แรงผลัก ($\Delta V$) เพื่อดูว่าถ้าดาวเทียมจุดจรวดหลบ จะมีเส้นทางเปลี่ยนไปอย่างไร โดยปรับหน่วยเป็น เมตร/วินาที:
- **In-track:** ไปข้างหน้า/ถอยหลัง
- **Cross-track:** ออกซ้าย/ขวา
- **Radial:** ดันขึน/ลง

### 1.6 การออกรายงาน (Report Generation)
เมื่อวิเคราะห์เสร็จ เลื่อนลงไปล่างสุดที่แถบ **"ส่งออกข้อมูล (Export Data)"** จะมีปุ่ม:
- **โหลดรายงาน .CSV:** เพื่อนำพิกัดทั้งหมดไปวิเคราะห์ต่อใน Excel
- **โหลดรายงาน .PDF:** ระบบจะนำกราฟและข้อมูลทั้งหมดมาจัดรูปเล่มเป็นไฟล์ PDF ระดับมืออาชีพ

## 💻 3. คำอธิบายโค้ดอย่างละเอียด (Detailed Code Walkthrough)

### 3.1 `app.py` (ไฟล์เริ่มต้นและจัดการ UI หลัก)
นี่คือหัวใจของแอปพลิเคชันที่สร้างขึ้นด้วย Streamlit:
- **บรรทัดที่ 1-50 (Imports):** นำเข้าฟังก์ชันต่างๆ ทั้งโมดูลคำนวณวงโคจร (core) และตัววาด UI (ui) รวมถึงไลบรารี `datetime` และ `pandas`.
- **`st.set_page_config`:** ตั้งค่าหน้าตาของเว็บ เช่น หน้าจอแบบกว้าง (wide)
- **Sidebar Logic (แถบด้านซ้าย):**
  - มีการสร้างปุ่มกด 2 ภาษา (`st.session_state.lang`) เพื่อสลับ `en` และ `th`.
  - มีการใช้ `st.radio` สำหรับเลือก **App Mode** (โหมด 1-on-1 หรือ Scanner) 
  - การกำหนดเวลาผ่าน `st.date_input` และ `st.time_input` แล้วนำมารวมกันเป็นตัวแปรชื่อ `start_datetime` พร้อมใส่ `timezone.utc`.
  - ตรวจสอบความสดใหม่ของข้อมูลผ่าน TLE Badges: `get_tle_age(sat1)`.
- **Main Analysis Loop (ระบบรับคำสั่งประมวลผล):** 
  - `if run_analysis:` จะเป็นการตรวจสอบเมื่อผู้ใช้งานกดปุ่ม "เริ่มการวิเคราะห์" 
  - หาก `app_mode` เป็นโหมดสแกน ระบบจะสั่ง `from core.scanner import run_catalog_scan` เเล้วสร้าง DataFrame จากนั้นใช้ `st.stop()` เพื่อหยุดโค้ดไม่ให้รันงานของ 1-on-1 ต่อ.
  - ในฟังก์ชัน 1-on-1 ระบบจะโหลด TLE -> วนเข้าหาเวลาผ่าน `generate_time_array` -> สร้าง progress bar รับค่า `compute_positions` และ `compute_itrf_positions` -> ส่งเข้ากราฟ `st.plotly_chart` ทั้ง 4 กราฟ.
  - ข้อมูลสุดท้ายเรียก `export_df.to_csv` และ `generate_pdf_report(..)` จากนั้นแสดงปุ่ม `st.download_button`.

### 3.2 `core/tle_fetcher.py` (ดึงพารามิเตอร์วงโคจร)
สำหรับจัดการข้อมูล Two-Line Elements (TLE):
- **`get_available_satellites()`**: ดึงชื่อดาวเทียมทั้งหมดที่มีใน `SATELLITE_CATALOG` 
- **`get_satellite(name, ts)`**: ใช้ `skyfield.api.EarthSatellite` เพื่อแปลงบรรทัด 1 กับ 2 ให้กลายเป็นวัตถุดาวเทียมที่พร้อมสำหรับการคำนวณทางฟิสิกส์.
- **`get_tle_age(satellite)`**: ทำการเรียกค่า `satellite.epoch.utc_datetime()` เพื่อตรวจสอบดูว่าแคตตาล็อก TLE ที่บันทึกไว้นั้น ถูกปล่อยออกมาเมื่อกี่วันก่อนหน้า โดยเอาไปลบกับ `datetime.now(timezone.utc)`.

### 3.3 `core/orbit_calculator.py` (ระบบคำนวณทางฟิสิกส์เบื้องต้น)
- **`generate_time_array`**: ตอนนี้ถูกอัปเกรดให้รับ `start_time` ได้. ฟังก์ชันนี้จะแปลงวันที่ผ่าน `timedelta(minutes=step)` แล้วบรรจุใส่ `ts.utc(...)` ของ Skyfield คืนค่าออกเป็น List ข้อมูลเวลา.
- **`compute_positions / compute_velocities`**: ลูปเข้าไปดึงค่า x, y, z (กม.) ของดาวเทียมในแต่ละเวลาด้วยเฟรม ECI (Earth-Centered Inertial - เฟรมไม่หมุนตามโลก).
- **`compute_itrf_positions`**: การแสดงดาวเทียมไปพร้อมกับลูกโลก 3D ต้องใช้พิกัดที่ชื่อ ITRF (Earth-Centered Earth-Fixed - เฟรมหมุนรอบแบบโลก). ฟังก์ชันนี้เรียกคำสั่งพิเศษ `sat.at(ti).itrf_xyz().km` เพื่อให้ได้พิกัดตามตำแหน่งภูมิศาสตร์.

### 3.4 `core/proximity_analysis.py` (ระบบจำลองเหตุการณ์พุ่งชน)
เป็นโมดูลที่ซับซ้อนที่สุดในระบบ:
- **`compute_relative_positions`**: แปลงพิกัดแบบปกติตามโลก (ECI) ให้กลายเป็น RIC Frame (Radial, In-track, Cross-track). RIC เฟรมทำให้เวลายืนอยู่บนอวกาศ จะเห็นดาวเทียมอีกดวงหนึ่งเคลื่อนที่สัมพันธ์กับเราอย่างไร 
- **`compute_distances`**: หาระยะห่างทั้งหมดแบบ Euclidean Math (`np.sqrt(dx^2 + dy^2 + dz^2)`).
- **`compute_collision_probability`**: ใช้การคำนวณสมการแบบกระจายตัว Alfano/Chan (Gaussian distribution) เพื่อหาเปอร์เซ็นต์โอกาสการชน (`pc`) เมื่อให้ความคลาดเคลื่อน `position_uncertainty`.
- **`apply_cw_maneuver(...)`**: สมบูรณ์แบบด้วย **Clohessy-Wiltshire (CW) Equations**. เมื่อได้แรงดัน $\Delta V_x, \Delta V_y, \Delta V_z$ มา ฟังก์ชันจะหารความเร็วเฉลี่ย (`n = v0 / r0`) และสร้างสมการตรีโกณมิติเพื่อหักเหเส้นทางการเคลื่อนที่ที่สัมพันธ์ให้เป็นแบบโค้ง ทำให้สามารถคำนวณระยะหลบหลีกใน 1 หรือ 2 การกระทำได้.

### 3.5 `core/scanner.py` (ค้นหา 1-to-All)
- รับดาวเทียมหลัก 1 ดวง -> วนลูปผ่านดาวเทียมที่เหลือใน `SATELLITE_CATALOG` เสมือนการรัน 1-on-1 หลายๆครั้ง
- ระบบจะกรองจับ `np.argmin(distances)` หาระยะที่พุ่งเข้าใกล้ที่สุดของทุกคู่วัตถุ แล้วบันทึกใส่ `threats`
- ผลลัพธ์สุดท้ายจะทำ `.sort(key=...)` เพื่อจัดอันดับจากอันตรายมากที่สุดเรียงลำดับไปน้อยที่สุด ก่อนส่งกลับให้ `app.py`.

### 3.6 `core/report_generator.py` (สร้างรายงานอัติโนมัติแบบ FPDF)
- จำเป็นต้องมีไลบรารี `fpdf2` (ตัวจัดการเลย์เอาต์กระดาษ) และ `kaleido` (แบคเอนด์ของ Plotly ที่ใช้แปลงกราฟเป็นรูปภาพ).
- **`generate_pdf_report(...)`**: 
  1. สร้างออบเจกท์ PDF
  2. จัดเรียง Title / โอกาสเกิดการชน / ระยะเวลา
  3. สั่งกราฟทั้ง 4 อัน (Distance, Heatmap, RIC 3D, ITRF 3D) ให้ทำการเซฟตัวเองลงเครื่องแบบชั่วคราว (`.write_image("temp_path.png")`)
  4. ใช้คำสั่ง `pdf.image(...)` เพื่อโยนรูปภาพลงไปในเอกสารทีละบรรทัด 
  5. `os.remove` เพื่อลบรูปลบออกหลังได้เอกสาร .pdf แบบ Buffer เป็นที่เรียบร้อย

### 3.7 `ui/charts.py` และ `ui/alerts.py` (ระบบภาพ Visualization)
- คุม UI ทั้งหมดเพื่อไม่ให้ `app.py` ดูรกเกินไป
- `ui/alerts.py`: สร้างป้ายประกาศเตือนภัยเช่นสีแดงตอนชน พร้อมตารางอัปเดต Telemetry (ความเร็ว, ตำแหน่ง).
- **`create_3d_relative_orbit(perturbed_ric)`**: ตัวกราฟลูปล่าสุด สามารถรับค่า `perturbed_ric` เส้นทางโดนหลบหลีกได้. หากมีการโดนหลบ โค้ดท่อน `if perturbed_ric is not None:` จะเพิ่มเส้นสีเขียว (Green dotted/solid line) ไปในกราฟเพื่อให้เทียบความแตกต่าง.
- **`create_3d_earth_view`**: ใช้ฟังก์ชัน `np.outer(np.cos(u), np.sin(v))` วาดลูกโลก 3 มิติที่เป็นสมการ Surface พร้อมกับพลอตจุด ITRF ของดาวเทียมใส่ลงไป ทำให้ผู้ใช้สามารถหมุนดูพื้นที่โลกเบื้องล่างได้อย่างไหลลื่น
